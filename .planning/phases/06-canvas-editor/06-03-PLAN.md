    Step 2: Update `frontend/src/components/editor/FabricCanvas.tsx` to react to store changes:

    The canvas must reactively update when the zustand store changes. Use `useEditorStore.subscribe()` or read store values in a useEffect with dependencies.

    Add these reactive behaviors:
    a. **Text content sync**: When `textOverrides` change in the store (from right panel inputs), find the corresponding IText object on canvas by `data.areaId` and update its `text` property. Then call `canvas.renderAll()`.
    b. **Text style sync**: When fontSize, color, fontFamily, fontWeight change in textOverrides, update the corresponding IText properties (`fontSize`, `fill`, `fontFamily`, `fontWeight`) and `canvas.renderAll()`.
    c. **Badge toggle sync**: When `badgeEnabled` changes, show/hide the badge group on canvas. Find badge objects by `data.type === 'badge'` and set `visible` property. Call `canvas.renderAll()`.
    d. **Badge text sync**: When `badgeText` changes, update the Text object inside the badge Group.
    e. **Marketplace dimension sync**: When `canvasWidth`/`canvasHeight` change (from marketplace switch), recreate the canvas contents at new dimensions. This means:
       - Update canvas dimensions: `canvas.setWidth(newWidth); canvas.setHeight(newHeight);`
       - Resize the background rect to match
       - Reposition objects proportionally (scale all x/y/width/height by newWidth/oldWidth ratio)
       - OR simpler: clear and rebuild canvas from template config at new dimensions
    f. **Font rendering on canvas**: When an IText has its fontFamily changed, fabric.js needs the font to be loaded first. Since `loadAllFonts()` is called on mount, fonts should be ready. But add a safety check: if `document.fonts.check(`${weight} ${size}px "${family}"`)` is false, load it first.

    For the store-to-canvas sync, use a pattern like:
    ```typescript
    useEffect(() => {
      const canvas = canvasRef.current;
      if (!canvas) return;

      // Find all IText objects and update from store
      canvas.getObjects().forEach((obj) => {
        if (obj.data?.type === 'text' && obj.data?.areaId) {
          const override = textOverrides[obj.data.areaId];
          if (override && obj instanceof IText) {
            obj.set({
              text: override.content,
              fontSize: override.fontSize,
              fill: override.color,
              fontFamily: override.fontFamily,
              fontWeight: override.fontWeight === 'bold' ? 'bold' : 'normal',
            });
          }
        }
      });
      canvas.renderAll();
    }, [textOverrides]);
    ```

    Similar patterns for badgeEnabled, badgeText, canvasWidth, canvasHeight.

    Step 3: Canvas-to-store sync (reverse direction):
    When user edits text directly on canvas (double-click IText), the `text:changed` event fires. This must update the store:

    canvas.on('text:changed', (e) => {
      const target = e.target;

    When user moves/resizes the product image on canvas, update the product position in overlay_data. This is handled via `object:modified` event on the product image object.

    Step 4: Selection highlighting:
    When user clicks a text input in the right panel (which calls `setSelectedObject('text', areaId)`), find the corresponding IText on canvas and call `canvas.setActiveObject(textObj)`. This highlights it on canvas.

    To do this, add a useEffect that watches `selectedTextAreaId`:
    ```typescript
    useEffect(() => {
      const canvas = canvasRef.current;
      if (!canvas || !selectedTextAreaId) return;
      const textObj = canvas.getObjects().find(
        (obj) => obj.data?.type === 'text' && obj.data?.areaId === selectedTextAreaId
      );
      if (textObj) {
        canvas.setActiveObject(textObj);
        canvas.renderAll();
      }
    }, [selectedTextAreaId]);
    ```
