Functions to implement:
- `get_presigned_put_url(bucket: str, object_name: str, expires: timedelta = timedelta(minutes=5)) -> str` -- generates presigned PUT URL for direct client upload. Uses `client.presigned_put_object(bucket, object_name, expires=expires)`. Presigned URL expires in 5 minutes (Claude's Discretion).
- `get_presigned_get_url(bucket: str, object_name: str, expires: timedelta = timedelta(hours=1)) -> str` -- generates presigned GET URL for viewing/downloading. Uses `client.presigned_get_object(bucket, object_name, expires=expires)`. Expires in 1 hour.
- `upload_bytes(bucket: str, object_name: str, data: bytes, content_type: str) -> None` -- uploads bytes directly to MinIO. Creates BytesIO wrapper, calls `client.put_object(bucket, object_name, BytesIO(data), len(data), content_type=content_type)`.
- `download_bytes(bucket: str, object_name: str) -> bytes` -- downloads object from MinIO as bytes. Uses `client.get_object(bucket, object_name)`, reads response, closes it.
- `delete_object(bucket: str, object_name: str) -> None` -- deletes object from MinIO. Uses `client.remove_object(bucket, object_name)`.
- `object_exists(bucket: str, object_name: str) -> bool` -- checks if object exists. Uses `client.stat_object()` in try/except, returns True/False.

**backend/app/services/images.py** -- Image validation and processing (per D-01, D-02, D-06):

```python
from io import BytesIO
from PIL import Image as PILImage

# Magic bytes for supported formats
MAGIC_BYTES = {
    "image/jpeg": [b"\xff\xd8\xff"],
    "image/png": [b"\x89PNG\r\n\x1a\n"],
    "image/webp": [b"RIFF"],  # followed by 4 bytes then "WEBP"
}

MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
MIN_DIMENSION = 200
MAX_DIMENSION = 8000
RESIZE_THRESHOLD = 4000  # auto-resize if larger side exceeds this
```

Functions to implement:
- `validate_magic_bytes(file_header: bytes) -> str | None` -- reads first 12 bytes, checks against MAGIC_BYTES dict. For WebP, check bytes[0:4] == b"RIFF" AND bytes[8:12] == b"WEBP". Returns detected MIME type string ("image/jpeg", "image/png", "image/webp") or None if no match. Per D-02: prevents renamed .exe files from passing validation.
- `validate_file_size(size: int) -> bool` -- returns True if size <= MAX_FILE_SIZE (10MB). Per D-01.
- `get_image_dimensions(data: bytes) -> tuple[int, int]` -- opens image with `PILImage.open(BytesIO(data))`, returns (width, height). Sets `PILImage.MAX_IMAGE_PIXELS = 178956970` (4 * default) to handle large images without decompression bomb protection triggering on legitimate product photos.
- `validate_dimensions(width: int, height: int) -> tuple[bool, str]` -- checks min 200x200 and max 8000x8000. Returns (True, "") or (False, error_message). Per D-01.
- `resize_if_needed(data: bytes, content_type: str) -> tuple[bytes, int, int, bool]` -- if max(width, height) > RESIZE_THRESHOLD (4000px), resize proportionally so longest side = 4000px using PILImage.LANCZOS resampling. Save back to bytes with same format. Returns (image_bytes, new_width, new_height, was_resized). Per D-06. For JPEG, save with quality=95. For PNG, save as PNG. For WebP, save with quality=95.
- `get_extension_from_mime(mime_type: str) -> str` -- returns "jpg" for image/jpeg, "png" for image/png, "webp" for image/webp.

Note on presigned URL flow (per D-03): The upload is a 2-step process:
1. POST /api/images/upload -- backend validates credits, generates image_id + presigned PUT URL, creates Image record with status="uploading"

This is cleaner than multipart upload because MinIO handles the large file transfer directly, and the backend never buffers 10MB files in memory.

**Endpoint 1: POST /api/images/upload** (per D-01, D-03, D-04, D-05, UPLD-01, UPLD-03, UPLD-04):

Two-step presigned URL flow:
- Accepts `multipart/form-data` with `file: UploadFile` parameter.
- Read the file content into memory (up to 10MB check first).
- Step 1: Validate file size <= 10MB. If over, return 400 with error "Файл слишком большой. Максимум 10 МБ".
- Step 2: Read first 12 bytes, call `image_svc.validate_magic_bytes()`. If None, return 400 with error "Неподдерживаемый формат. Допустимы JPG, PNG, WebP" (per D-02, UPLD-02).
- Step 3: Get image dimensions via `image_svc.get_image_dimensions(file_data)`. Validate with `image_svc.validate_dimensions()`. If invalid, return 400 with error message.
- Step 4: Check `user.credits_remaining > 0`. If 0, return 403 with error "Нет доступных карточек. Оформите подписку" (per D-05, UPLD-04).
- Step 5: Auto-resize if needed via `image_svc.resize_if_needed(file_data, detected_mime)`. This returns (data, width, height, was_resized) (per D-06, UPLD-10).
- Step 6: Generate `image_id = uuid.uuid4()`, determine extension from MIME via `image_svc.get_extension_from_mime(detected_mime)`.
- Step 7: Compute object_name = `{user.id}/{image_id}.{ext}` (per D-04).
- Step 8: Upload the (potentially resized) image bytes to MinIO: `minio_svc.upload_bytes(minio_svc.BUCKET_ORIGINALS, object_name, final_data, detected_mime)`.
- Step 9: Generate presigned GET URL for viewing: `original_url = minio_svc.get_presigned_get_url(minio_svc.BUCKET_ORIGINALS, object_name)`.
- Step 10: Create Image record in DB: `Image(id=image_id, user_id=user.id, original_url=object_name, original_filename=file.filename or "upload", original_size=len(final_data), original_width=width, original_height=height, status="uploaded")`. Note: `original_url` stores the MinIO object path (not the presigned URL, which expires). The presigned GET URL is generated on read.
- Step 11: Commit and return 201 with ImageResponse populated from the record. For the response, set `original_url` to the presigned GET URL (not the storage path) so the client can display a preview.

Note: This is a SIMPLIFIED presigned flow -- the backend receives the file via multipart (FastAPI UploadFile), validates it server-side, and uploads to MinIO itself. This is more secure than pure client-side presigned PUT because the server validates magic bytes before anything reaches storage. The file data flows: client -> backend (validation) -> MinIO. This avoids the complexity of a confirm endpoint while keeping magic byte validation server-side. For 10MB max files, this memory usage is acceptable.

IMPORTANT: Store the MinIO object path (e.g., "user_id/image_id.jpg") in Image.original_url, NOT the presigned URL. Presigned URLs expire. Generate fresh presigned URLs when serving the image to the client.

```python
async def _get_image_or_404(image_id: uuid.UUID, user: User) -> Image:
    async with AsyncSessionLocal() as session:
        result = await session.execute(
            select(Image).where(Image.id == image_id, Image.user_id == user.id)
        )
        image = result.scalar_one_or_none()
        if image is None:
            raise HTTPException(status_code=404, detail="Изображение не найдено")
        return image
```
