Define TypeScript interfaces matching the backend schemas:

```typescript
export interface DashboardStats {
  plan: string;
  credits_remaining: number;
  credits_total: number;
  renders_this_month: number;
  renders_total: number;
  subscription_expires_at: string | null;
}

export interface RenderItem {
  id: string;
  image_id: string;
  template_id: string;
  marketplace: string;
  output_url: string | null;
  output_width: number;
  output_height: number;
  created_at: string;
}

interface RenderListResponse {
  renders: RenderItem[];
  total: number;
  limit: number;
  offset: number;
}
```

Import the shared axios instance: `import api from "../lib/api"` (NOT raw axios -- this has the JWT interceptor).

Create API functions:
1. `fetchDashboardStats()` -- GET /dashboard/stats, returns DashboardStats.
2. `fetchRenders(limit: number, offset: number)` -- GET /renders?limit={limit}&offset={offset}, returns RenderListResponse.
3. `deleteRender(renderId: string)` -- DELETE /renders/{renderId}, returns void. This is NOT wrapped in useQuery (mutation).
4. `downloadRender(renderId: string)` -- GET /renders/{renderId}/download, returns { download_url: string, filename: string }. Used imperatively (not a hook).

Create React Query hooks:
1. `useDashboardStats()` -- useQuery with queryKey: ["dashboard", "stats"], staleTime: 30_000 (30 seconds).
2. `useRenders(limit: number, offset: number)` -- useQuery with queryKey: ["renders", limit, offset], keepPreviousData: true (so grid doesn't flash empty on page change).

Export all types, hooks, and functions.

Pattern: follow the same style as frontend/src/api/templates.ts -- import from "@tanstack/react-query", use useQuery, export the hook function.

Use useRenders(PAGE_SIZE, offset) hook with PAGE_SIZE = 20.

Grid layout: grid grid-cols-2 lg:grid-cols-4 gap-4. Each card:
- Thumbnail image: output_url as src, aspect-ratio 3/4 for WB (900x1200), 1/1 for Ozon (1200x1200). Use object-cover, rounded-lg. If output_url is null, show a gray placeholder.
- Marketplace badge: absolute positioned top-right corner. Colors per CONTEXT.md specifics: WB = purple (bg-purple-500), Ozon = blue (bg-blue-500), YM = yellow (bg-yellow-500). White text, uppercase abbreviation (WB, OZ, YM), text-xs font-bold px-2 py-0.5 rounded-bl-lg.

Download button: on click, call downloadRender(id), then window.open(download_url) or create a temporary <a> element with download attribute. Delete button: on click, show window.confirm("Удалить карточку?"), if confirmed call deleteRender(id), then invalidate the renders query via queryClient.invalidateQueries({queryKey: ["renders"]}) and also invalidate ["dashboard", "stats"] to update render counts.

Use `useQueryClient()` from @tanstack/react-query for invalidation after delete.

**3. Load More Pagination (per D-06, DASH-04):**
Track offset in component state (useState, starts at 0).
If total > offset + PAGE_SIZE, show a "Загрузить ещё" button (w-full py-3, border, rounded-lg, text-gray-700, hover:bg-gray-50).
On click: setOffset(prev => prev + PAGE_SIZE). The useRenders hook will re-fetch with new offset.
Note: this approach shows only one page at a time. For a "load more" that appends: keep a local state array of all loaded renders, and on "load more" fetch next page and append to the array. Simpler: use useInfiniteQuery from @tanstack/react-query. Pick the simpler useState approach: keep `allRenders` state, fetch next page, concat.

Alternative simpler approach: use a single state `page` (starting at 1). Fetch with offset = (page - 1) * PAGE_SIZE. Keep all previously loaded items in a ref/state. On "load more", increment page, fetch new page, append to existing renders. Show "Загрузить ещё" if renderedCount < total.
