**backend/alembic/env.py** -- Async migration runner (from Research Pattern 3, with Pitfall 4 fix):
- Import `asyncio` from stdlib
- Import `fileConfig` from `logging.config`
- Import `pool` from `sqlalchemy`
- Import `async_engine_from_config` from `sqlalchemy.ext.asyncio`
- Import `context` from `alembic`
- Import `Base` from `app.models` (this import triggers all model imports via __init__.py, making Alembic see all tables)
- Read DATABASE_URL from environment variable `os.environ.get("DATABASE_URL")` and if present, override `config.set_main_option("sqlalchemy.url", ...)` with the async driver prefix replacement (`postgresql://` -> `postgresql+asyncpg://`). This handles Pitfall 4.
- Set `target_metadata = Base.metadata`
- Implement `run_migrations_offline()` for SQL generation mode
- Implement `do_run_migrations(connection)` that calls `context.configure(connection=connection, target_metadata=target_metadata)` with `context.begin_transaction()` and `context.run_migrations()`
- Implement `async run_async_migrations()` that creates engine via `async_engine_from_config`, connects, runs `do_run_migrations` via `connection.run_sync()`, then disposes
- Implement `run_migrations_online()` that calls `asyncio.run(run_async_migrations())`
- Route to offline or online based on `context.is_offline_mode()`
