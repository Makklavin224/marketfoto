- `redis`: image `redis:7-alpine`, command `redis-server --maxmemory 256mb --maxmemory-policy noeviction --appendonly yes`. Healthcheck: `redis-cli ping`, interval 5s, timeout 3s, retries 5. Memory limit 300M.
- `minio`: image `cgr.dev/chainguard/minio:latest` (per D-05, NOT minio/minio which is discontinued). Command: `server /data --console-address ":9001"`. Env: `MINIO_ROOT_USER=${MINIO_USER}`, `MINIO_ROOT_PASSWORD=${MINIO_PASSWORD}`. Volume: `miniodata:/data`. Healthcheck: `wget -q --spider http://localhost:9000/minio/health/live || exit 1` (use wget not curl -- Chainguard image may not have curl, per Pitfall 2). Interval 10s, timeout 5s, retries 5, start_period 10s. Memory limit 512M.
- `minio-init`: image `quay.io/minio/mc:latest`. depends_on minio (service_healthy). entrypoint `/bin/sh`. command creates 3 buckets: originals, processed, rendered using `mc alias set myminio http://minio:9000 $${MINIO_USER} $${MINIO_PASSWORD}; mc mb --ignore-existing myminio/originals; mc mb --ignore-existing myminio/processed; mc mb --ignore-existing myminio/rendered; exit 0;`. Env: `MINIO_USER=${MINIO_USER}`, `MINIO_PASSWORD=${MINIO_PASSWORD}`.
- `backend`: build context `./backend`. Command: `sh -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000 --proxy-headers"` (per Research recommendation -- run migrations in entrypoint). env_file: `.env`. depends_on: postgres (service_healthy), redis (service_healthy), minio (service_healthy). Healthcheck: `curl -f http://localhost:8000/api/health`, interval 10s, timeout 5s, retries 5, start_period 15s. Memory limit 512M.
- `worker`: build context `./worker`. Command: `rq worker --url redis://redis:6379 --max-jobs 100` (per D-04). env_file: `.env`. depends_on: redis (service_healthy), minio (service_healthy). Memory limit 2G (per D-04 and INFRA-06). restart: unless-stopped.
- `frontend`: build context `./frontend`. Healthcheck: `wget -q --spider http://localhost:3000 || exit 1`, interval 10s, timeout 5s, retries 3. Memory limit 256M.
- `nginx`: image `nginx:alpine`. Ports: 80:80, 443:443. Volumes: `./nginx/nginx.conf:/etc/nginx/nginx.conf:ro`, `certbot-webroot:/var/www/certbot:ro`, `certbot-certs:/etc/letsencrypt:ro`. depends_on: backend (service_healthy), frontend (service_healthy). Memory limit 128M. restart: unless-stopped.

Named network: `default` with name `marketfoto`.
Named volumes: `pgdata`, `miniodata`, `certbot-webroot`, `certbot-certs`.

**worker/Dockerfile** (per D-02 -- pre-download birefnet-general at build time):
```dockerfile
FROM python:3.12-slim AS base
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*
COPY backend/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY backend/app ./app
# Pre-download birefnet-general model (~1.5GB baked into image layer) per D-02
RUN python -c "from rembg import new_session; new_session('birefnet-general')"
# Pre-warm numba JIT compilation to avoid first-request latency
ENV NUMBA_CACHE_DIR=/tmp/numba_cache
RUN python -c "\
from rembg import remove, new_session; \
from PIL import Image; \
import io; \
session = new_session('birefnet-general'); \
img = Image.new('RGB', (100, 100), 'white'); \
buf = io.BytesIO(); \
img.save(buf, 'PNG'); \
buf.seek(0); \
remove(buf.getvalue(), session=session); \
print('Model pre-warmed successfully')"
CMD ["rq", "worker", "--url", "redis://redis:6379", "--max-jobs", "100"]
```
